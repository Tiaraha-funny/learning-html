Tips of HTML

This page will be covered of three kinds of advices
  • The way how to write HTML quicker
  • The tools that you can use
  • The way how to make sure that your code is in a high quality

The way how to write HTML quicker

  1. Using keyboard : Try to not touch mouse.

    Use: ctrl + shift + P and you will find:
    - Toggle word wrap : (to make the letter not to go straight)
    - Format document : (to make a good indent)
    - Open preview to the side: to see the preview of your document
    - Create Branch from visual studio code
    - Making color of the theme ...etc.

  2. Use small arrow key :
    - ctrl + end : to come at the end of the document.
    - ctrl + home : to come at the very begining.
    - ctrl + pg up and pg down : to make the document turn right or left.
    - Select and ctrl + press D : to edit something together.

  3. Use ctrl tab : to come very quickly in the other browser, and ctrl + shift + tab to return.

  4. Use ctrl + shift C to copy and ctrl + shift V to paste.

  5. - Use ctrl + T to open new tab in the browser
    - Use ctrl + shift N to open new vsial studio code
    - Use ctrl + N to open a new file in visual studio
    - Use ctrl + Z to undo and ctrl + Y to redo.

  6. To know more about the keyboard visit Keyboard shortcut

The tools that you can use

  1. You need git bush to start working.

  2. You need visual studio to work on:

    - New file
    - New folder

The way how to make sure that your work is in high quality

  1. - Create a repo and name it as you like and add a README.md to make it better.
    - Copy and clone the SSH URL for this new repo in a directory on your computer, preferably in ~/code/html. make sure that it is SSH but not HTTPS.
    - Create a new branch named as you like it and push, and check it in the repo settings.
    - Add the URL for the website to the README.md file
    - And checkout master as a default development branch
    - Merging it to the new branch not master to publish it.

  2. Check your work on HTML5.validator.

  3. - Create new pull request
    - Send your pull request to a reviewer
